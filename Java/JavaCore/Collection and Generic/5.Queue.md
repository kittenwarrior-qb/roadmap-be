# Java Queue

> `Queue` là interface trong `java.util`, mở rộng từ `Collection`, hoạt động theo nguyên tắc **FIFO** (First-In-First-Out).

---

## 1. Đặc điểm chính

- Thêm phần tử ở **đuôi (tail)**, lấy/xoá ở **đầu (head)**.
- Các implementation phổ biến: `LinkedList`, `ArrayBlockingQueue`, `PriorityQueue`.
- **BlockingQueue**: không chấp nhận `null`, thread-safe, dùng cho Producer/Consumer.
- **Deque**: hàng đợi hai đầu — thêm/xoá được ở cả hai phía.
- `java.util` → **Unbounded** Queue | `java.util.concurrent` → **Bounded** Queue.

---

## 2. Các Methods

### Kế thừa từ `Collection`

| Method | Mô tả |
|--------|-------|
| `size()` | Số phần tử trong queue |
| `isEmpty()` | Kiểm tra queue rỗng không |
| `contains(Object o)` | Kiểm tra phần tử có tồn tại không |
| `clear()` | Xoá tất cả phần tử |
| `toArray()` | Chuyển queue thành mảng |
| `iterator()` | Trả về iterator |
| `removeAll(Collection c)` | Xoá các phần tử có trong collection c |
| `retainAll(Collection c)` | Giữ lại chỉ các phần tử có trong c |

### Đặc trưng của `Queue`

| Thao tác   | Ném Exception     | Trả về giá trị đặc biệt |
|-----------|-------------------|--------------------------|
| **Insert** | `add(e)` → `IllegalStateException` | `offer(e)` → `false` |
| **Remove** | `remove()` → `NoSuchElementException` | `poll()` → `null` |
| **Examine** | `element()` → `NoSuchElementException` | `peek()` → `null` |

---

## 3. Ví dụ cơ bản

```java
Queue<String> queue = new LinkedList<>();
queue.add("one");
queue.add("two");
queue.add("three");
queue.add("four");

System.out.println(queue);              // [one, two, three, four]
queue.remove("three");
System.out.println(queue);              // [one, two, four]
System.out.println(queue.size());       // 3
System.out.println(queue.contains("two")); // true
queue.clear();
```

---

## 4. Chuyển đổi Array ↔ Queue

```java
// Array → Queue
String[] nums = {"one", "two", "three"};
Queue<String> queue = new LinkedList<>();
Collections.addAll(queue, nums);
System.out.println(queue); // [one, two, three]

// Queue → Array
String[] strArray = queue.toArray(new String[queue.size()]);
System.out.println(Arrays.toString(strArray)); // [one, two, three]
```

## 5. Chi tiết từng thao tác

### Insert: `add()` vs `offer()`

```java
BlockingQueue<String> queue = new ArrayBlockingQueue<>(2);

queue.add("one");   // true
queue.add("two");   // true
queue.add("three"); // ❌ IllegalStateException: Queue full

queue.offer("one");   // true
queue.offer("two");   // true
queue.offer("three"); // false (không ném exception)
```

### Remove: `remove()` vs `poll()`

```java
Queue<String> queue = new LinkedList<>();
queue.offer("one");
queue.offer("two");

queue.remove(); // "one"
queue.remove(); // "two"
queue.remove(); // ❌ NoSuchElementException

queue.poll(); // "one"
queue.poll(); // "two"
queue.poll(); // null (không ném exception)
```

### Examine: `element()` vs `peek()`

```java
Queue<String> queue = new LinkedList<>();
queue.add("one");

queue.element(); // "one" — không xoá
queue.clear();
queue.element(); // ❌ NoSuchElementException

queue.peek();    // "one" — không xoá
queue.clear();
queue.peek();    // null (không ném exception)
```

## 6. Phân loại Queue

### Theo giới hạn capacity

| Loại | Mô tả | Package | Ví dụ |
|------|-------|---------|-------|
| **Unbounded** | Không giới hạn kích thước | `java.util` | `LinkedList` |
| **Bounded** | Phải chỉ định max size khi tạo | `java.util.concurrent` | `ArrayBlockingQueue` |

### Theo blocking

| Loại | Mô tả | Ví dụ |
|------|-------|-------|
| **BlockingQueue** | Block cho đến khi xong hoặc timeout | `ArrayBlockingQueue`, `LinkedBlockingQueue` |
| **Non-Blocking** | Không block | `LinkedList`, `PriorityQueue` |

---

## 7. BlockingQueue — Thao tác mở rộng

| Thao tác   | Ném Exception | Special Value | Block mãi | Timeout |
|-----------|:-------------:|:-------------:|:---------:|:-------:|
| **Insert** | `add(e)` | `offer(e)` | `put(e)` | `offer(e, time, unit)` |
| **Remove** | `remove()` | `poll()` | `take()` | `poll(time, unit)` |
| **Examine** | `element()` | `peek()` | – | – |

> `put(e)` / `take()` sẽ **block vô hạn** nếu queue đầy/rỗng.  
> `offer(e, time, unit)` / `poll(time, unit)` block nhưng có **timeout**.

---

## 8. Tổng kết nhanh

| Cần gì | Dùng gì |
|--------|---------|
| FIFO đơn giản | `LinkedList` |
| Bounded + thread-safe | `ArrayBlockingQueue` |
| Theo độ ưu tiên | `PriorityQueue` |
| Producer/Consumer pattern | `BlockingQueue` |
| Thêm/xoá 2 đầu | `Deque` (`ArrayDeque`) |
| Unbounded + thread-safe | `ConcurrentLinkedQueue` |
